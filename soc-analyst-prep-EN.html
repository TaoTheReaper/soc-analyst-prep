How SOC Triage Works

Triage is the process by which an L1 analyst evaluates an alert, determines its severity and legitimacy, and decides the next action. It is the core skill of any SOC L1.

Standard Triage Flow

1.
Alert Reception
The SIEM generates an alert with severity (LOW/MEDIUM/HIGH/CRITICAL)
2.
Contextualisation
Who is involved? Where does it how from? When? Is it normal for that user/system?
3.
Classification
False positive (FP), True positive (TP), False negative (FN — the worst)
4.
Action / Escalation
Close, block, isolate, reset credentials, or escalate to L2/L3
5.
Documentation
Log everything in the ticketing system (who, what, when, action taken)

Questions to ask on every alert

WHO — Which user or system? Is it privileged? Is it a service account?
WHAT — What action was performed? Does it match the role?
WHERE — From which IP? Internal or external? Country? Is it the usual workstation?
WHEN — Business hours or 3 AM? Public holiday?
HOW OFTEN — Isolated event or recurring pattern?
CONTEXT — Is there scheduled maintenance? A registered trip?
Golden rule:
Context first, action second. The only exception is active lateral movement or ongoing exfiltration — isolate immediately.
False negative (FN) = the worst possible mistake
The attack happened but the SIEM didn't see it. This is why detection rules need periodic review.

When to escalate to L2

Confirmed compromise
Lateral movement detected
Possible data exfiltration
Critical alert on core infrastructure
Unable to classify with certainty
Requires access or tools you don't have

MITRE ATT&CK Framework

The framework used by all SOCs to classify adversary tactics and techniques. You must be able to name it and explain it at a high level.

What it is in one sentence: MITRE ATT&CK is a public knowledge base of tactics, techniques and procedures (TTPs) used by real attackers, organised in a matrix. It is built from real-world attack observations and helps SOC teams understand how adversaries move, not just what fanno.

Why SOCs use it: Ogni alert del SIEM può essere mappato su ATT&CK per capire in quale fase dell'attacco ci si trova e what aspettarsi how mossa successiva.
Framework structure
• Tactics (14) — the "why" of the attack. E.g.: Initial Access, Lateral Movement, Exfiltration
• Techniques (~200) — il "how". Es: T1566 Phishing, T1078 Valid Accounts
• Sub-techniques (~400) — further detail. E.g.: T1566.001 Spearphishing Attachment
• Procedure — specific implementation by an APT group
How it is used in a SOC
• SIEM alert → map to ATT&CK technique → understand the phase
• Build detection rules starting from techniques
• Navigator: visual tool to see detection coverage
• Threat Intelligence: "This group uses T1059.003"
• Purple teaming: red team simulates techniques, blue team measures detection

The 14 tactics — left to right in the order of an attack

Phrase to use in an interview if asked about ATT&CK:
"MITRE ATT&CK lets me contextualise an alert within the attacker's kill chain. If I see a password spray alert, I know I'm in the Credential Access phase and the next likely step is Lateral Movement — so I start monitoring for that already."

Final Checklist

✅ Introduce yourself in 60s: technical background, projects, motivation
✅ Mention practical GitHub projects if asked about experience
✅ Think out loud on scenarios — never answer with a single word
✅ Malware: isolate first, do not power off
✅ Phishing: sender, headers, links, attachments, sandbox
✅ Ask about the SOC environment, vendors used and growth opportunities
✅ On salary: don't name a number first. Ask about the full package (shift bonuses, benefits)

Before any assessment: research the company on LinkedIn and their website. Understanding their structure, clients, vendors (SIEM, EDR) and own products shows genuine interest.
